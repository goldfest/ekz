# Exam Service Center SPA

Минимальное работоспособное веб-приложение для подготовки к экзамену.

## Тематика

**Учет клиентов сервисного центра.**

Приложение объединяет типовые требования из экзаменационных билетов: можно принять клиента, назначить сотрудника, оформить услугу, продлить срок заказа, завершить/отменить заказ, отредактировать данные и выполнить стандартные CRUD-операции.

## Реализовано

- SPA: React + JavaScript + Vite + Bootstrap.
- Backend: Java 21 + Spring Boot + Maven + Spring Security + JWT + Lombok.
- БД: PostgreSQL, работа через Spring Data JPA/Hibernate.
- Авторизация и регистрация по JWT.
- Роли: `ADMIN` и `USER`.
  - `ADMIN`: просмотр, добавление, редактирование, удаление.
  - `USER`: просмотр списков и карточек без изменения данных.
- 4 бизнес-сущности:
  - клиенты;
  - сотрудники;
  - услуги;
  - заказы.
- Many-to-many связь: **заказы ↔ услуги**.
- CRUD-операции, таблицы и карточки.
- Поиск, фильтрация и пагинация на backend и frontend.
- Адаптивный интерфейс на Bootstrap.
- Отчеты намеренно не реализованы, так как по текущему требованию они не нужны.

### 1. Backend

```bash
cd backend
./mvnw spring-boot:run
```

Backend будет доступен на `http://localhost:8080`.

Swagger UI: `http://localhost:8080/swagger-ui.html`

### 2. Frontend

```bash
cd frontend
npm install
npm run dev -- --host 127.0.0.1 --port 5173
```

Frontend будет доступен на `http://127.0.0.1:5173/`.

## Тестовые пользователи

Администратор:

- логин: `admin`
- пароль: `admin123`

Обычный пользователь:

- логин: `user`
- пароль: `user123`

Тестовые пользователи создаются автоматически при первом запуске backend.

